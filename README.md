<h2 id="comprehensive-feature-set" style="font-size: 48px; color: #ff4500; margin-top: 32px;">🛠️ Comprehensive Feature Set</h2>
<p style="font-size: 20px; color: #333333; line-height: 1.6;">
Below is a high-level overview of each core module and its capabilities:
</p>

<h3 style="font-size: 36px; color: #d90429; margin-top: 24px;">1. Link Extraction (<code>sp e &lt;url&gt;</code>)</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li><strong>Goal:</strong> Gather all <code>&lt;a href=""&gt;</code> links from a website.</li>
  <li><strong>Mechanism:</strong>
    <ul>
      <li>Sends GET request to target URL.</li>
      <li>Parses HTML with BeautifulSoup.</li>
      <li>Collects unique <code>href</code> attributes (absolute & relative).</li>
    </ul>
  </li>
  <li><strong>Output:</strong>
    <ul>
      <li>Creates folder <code>&lt;target-domain&gt;/</code>.</li>
      <li>Writes <code>links.txt</code> containing one URL per line.</li>
    </ul>
  </li>
</ul>

<h3 style="font-size: 36px; color: #d90429; margin-top: 24px;">2. SQL Injection Testing (<code>sp c &lt;url&gt;</code>)</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li><strong>Goal:</strong> Identify SQL injection vulnerabilities at a given parameter.</li>
  <li><strong>Mechanism:</strong>
    <ul>
      <li>Generates <strong>1,000+</strong> payloads via <code>utils.generate_sql_payloads()</code>.</li>
      <li>Appends each payload to the provided URL (e.g., <code>?id=1</code> → <code>?id=1' OR '1'='1</code>).</li>
      <li>Scans response for SQL error indicators (e.g., “SQL syntax”, “mysql_fetch”, “warning:”).</li>
      <li>Time-based checks by measuring response delays (for <code>SLEEP</code> payloads).</li>
    </ul>
  </li>
  <li><strong>Output:</strong>
    <ul>
      <li>Prints each vulnerable payload + URL to console.</li>
      <li>Writes <code>sqli_results.txt</code> in target folder, listing payloads and affected endpoints.</li>
    </ul>
  </li>
</ul>

<h3 style="font-size: 36px; color: #d90429; margin-top: 24px;">3. Admin Panel Discovery (<code>sp find &lt;url&gt;</code>)</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li><strong>Goal:</strong> Locate hidden or unlisted admin/login panels across a site.</li>
  <li><strong>Mechanism:</strong>
    <ul>
      <li>Loads <strong>1,000+</strong> paths from <code>utils.generate_admin_paths()</code>.</li>
      <li>Iterates sequentially or (optionally) multi-threaded.</li>
      <li>Sends HEAD/GET requests to <code>&lt;base_url&gt;/&lt;path&gt;</code>.</li>
      <li>Detects HTTP 200 OK or redirects to login.</li>
    </ul>
  </li>
  <li><strong>Output:</strong>
    <ul>
      <li>Prints “Found admin page: <code>&lt;full_url&gt;</code>” for each hit.</li>
      <li>Writes <code>admin_paths.txt</code> listing all discovered admin URLs.</li>
    </ul>
  </li>
</ul>

<h3 style="font-size: 36px; color: #d90429; margin-top: 24px;">4. Advanced Brute-Force Login (<code>sp brute &lt;login_url&gt;</code>)</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li><strong>Goal:</strong> Attempt brute-force against a login form with dynamic stealth.</li>
  <li><strong>Mechanism:</strong>
    <ul>
      <li>Loads default usernames from <code>utils.default_usernames()</code>: <code>[admin, root, admin1, admin2, rootadmin, …]</code>.</li>
      <li>Loads default passwords from <code>utils.default_passwords()</code>: <code>[password123, admin, 123456, …]</code>.</li>
      <li>Fetches <strong>1,000+</strong> random user-agents via <code>utils.generate_user_agents(1000)</code>.</li>
      <li>For each user-agent, for each username, for each password:
        <ul>
          <li>Sends POST request with <code>{'username': user, 'password': pwd}</code> and <code>{'User-Agent': &lt;ua&gt;}</code>.</li>
          <li>Checks if response contains any success keyword (“logout”, “dashboard”, “welcome”).</li>
        </ul>
      </li>
      <li>Stops early upon a successful login.</li>
    </ul>
  </li>
  <li><strong>Output:</strong>
    <ul>
      <li>On success: prints <code>Successful login: &lt;user&gt;:&lt;pwd&gt; (UA: &lt;ua&gt;)</code>, writes to <code>brute_results.txt</code>.</li>
      <li>If none succeed: prints <code>Brute force completed. No valid credentials found.</code>, still writes a summary log.</li>
    </ul>
  </li>
</ul>

<h2 id="troubleshooting--faqs" style="font-size: 48px; color: #ff4500; margin-top: 32px;">❓ Troubleshooting & FAQs</h2>

<h3 style="font-size: 32px; color: #d90429; margin-top: 24px;">1. <code>sp</code> command not recognized after installation?</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li>Ensure <code>pip install</code> or <code>python setup.py install</code> completed without errors.</li>
  <li>Verify that Python’s Scripts directory is in your PATH (e.g., <code>~/.local/bin</code> or <code>C:\PythonXY\Scripts</code>).</li>
  <li>Run <code>which sp</code> (Linux/macOS) or <code>where sp</code> (Windows) to confirm its location.</li>
</ul>

<h3 style="font-size: 32px; color: #d90429; margin-top: 24px;">2. ImportError or missing dependencies?</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li>Reinstall dependencies using:
    <pre style="background-color: #f8f9fa; padding: 12px; border-radius: 6px; font-size: 18px; color: #111111;">pip install -r requirements.txt</pre>
  </li>
  <li>If using a virtual environment, ensure it’s activated.</li>
</ul>

<h3 style="font-size: 32px; color: #d90429; margin-top: 24px;">3. CAPTCHA encountered during brute-force?</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li>Use <code>--captcha-bypass &lt;service_api_key&gt;</code> flag and configure <code>config/captcha.json</code> with your API key.</li>
  <li>The framework will automatically fetch the CAPTCHA image and call the solving service.</li>
</ul>

<h3 style="font-size: 32px; color: #d90429; margin-top: 24px;">4. HTML/PDF report generation failing?</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li>Ensure the required libraries are installed:
    <pre style="background-color: #f8f9fa; padding: 12px; border-radius: 6px; font-size: 18px; color: #111111;">pip install jinja2 weasyprint</pre>
  </li>
  <li>Verify that the <code>logs/</code> directory has write permissions.</li>
</ul>

<h3 style="font-size: 32px; color: #d90429; margin-top: 24px;">5. Tor support not working?</h3>
<ul style="font-size: 20px; color: #333333; line-height: 1.6;">
  <li>Ensure Tor service is running: <code>sudo service tor start</code> (Linux) or <code>brew services start tor</code> (macOS).</li>
  <li>Check that <code>localhost:9050</code> is open as a SOCKS5 proxy: <code>netstat -tnlp | grep 9050</code>.</li>
</ul>
